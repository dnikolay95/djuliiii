# Этап 4. REST API для просмотра данных (done)

## Цель
Выдать данные для админки через REST.

## Задачи
- GET /api/users: список с счетчиками поздравлений по каждому пользователю.
- GET /api/greetings: список поздравлений (пагинация, фильтры по пользователю).
- GET /api/messages: список сообщений (пагинация, фильтры по пользователю и типу).
- GET /api/stats: total greetings, total users, total messages, топ пользователей по поздравлениям.

## Результат проверки
- В браузере после логина таблицы users/greetings/messages отображают данные.
- Счетчики совпадают с БД.
