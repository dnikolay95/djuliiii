# Этап 6. Realtime без перезагрузки (done)

## Цель
Обновление админки "на лету" через WebSocket.

## Задачи
- Добавить WS /ws с авторизацией (токен/кука).
- При записи в БД отправлять события: user_upserted, message_received, greeting_sent (и stats_updated опционально).
- На фронте: подключение к WS после логина, обновление таблиц и счетчиков, переподключение при разрыве.

## Результат проверки
- Админка открыта. В другой вкладке пишем боту или жмем поздравление.
- Новые записи и счетчики обновляются без F5.
